Convert .xsd file to JAVA Classes

XSD (XML Schema Definition) file describes the structure of an XML document. It contains details about the elements, attributes, and data types of a document in an XML.

For instance, consider we have an XML file that needs to be used in JAVA. We can get the details of each tag using its corresponding schema definition (.xsd file) to create JAVA classes.

In this article, I will share the Maven plugin I used to generate JAVA classes for an XML file using its .xsd file.

I am using JAVA 11 and Maven.

Use the Maven plugin jaxb2-maven-plugin as below in pom.xml file:

<plugin>
    <groupId>org.codehaus.mojo</groupId>
    <artifactId>jaxb2-maven-plugin</artifactId>
    <version>1.5</version>
    <executions>
    <execution>
     <id>jaxb-test</id>
     <goals>
      <goal>xjc</goal>
     </goals>
     <configuration>
      <schemaDirectory>${project.basedir}/src/main/resources/</schemaDirectory>
      <packageName>model</packageName>
      <schemaFiles>pmml-4-4.xsd</schemaFiles>
      <outputDirectory>${project.build.directory}/generated-sources/JAVAClassesOfXSDFile</outputDirectory>
     </configuration>
    </execution>
    </executions>
   </plugin>

Provide the below details as per your requirement:

  • <schemaDirectory> — The directory location of .xsd file
  • <schemaFiles> — .xsdfile name with extension
  • <outputDirectory> — The directory location where JAVA classes are to be created
  • <packageName> — parent package for JAVA classes

runmvn clean install

It will generate the classes under the package: target/generated-sources/JAVAClassesOfXSDFile/model

Note: we need to update the XSD file for any duplicate element name in any document (or) we can use .xjb file to resolve any conflict

There are other plugins also available for this conversion. I have included two of them in the GitHub repo.

For illustration, I have taken a pmml-4.4.xsd file from PMML 4.4 — Anamoly Detection Model.
