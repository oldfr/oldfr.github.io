What is Aspect Oriented Programming (AOP) in Spring

Aspect Oriented Programming is a pluggable way to do something before, after, or around a method call. Consider that you have an existing project and you need to add some functionality before each method in a specific class called. This could be adding logger messages, for example.

In the spring official documentation, you can find many terminologies used. Those include:

  1. Aspect: The class that handles code to intercept method calls in a specified class. Use @Aspect on class level
  2. Join Point: It is the method to be intercepted.
  3. Advice: Action that could be taken on a method. It could be — before, after, around, afterReturning or afterThrowing any exception from the JointPoint(method)
  4. Pointcut: The predicate that matches the join points. It could be all methods in a class. By default, spring uses the AspectJ pointcut expression language.
  5. Introduction: Spring AOP lets you introduce new interfaces (and a corresponding implementation) to any advised object.
  6. Target Object: An object being advised by one or more aspects.
  7. AOP proxy: It is an object created by the AOP framework in order to implement the aspect contracts
  8. Weaving: Linking aspects with other application types or objects to create an advised object. Spring AOP, performs weaving at runtime.

Types of advices in AOP:

  • Before advice: The method that will run before the join point (method).
  • After advice: The method that will run after execution of join point.
  • After Returning advice: The method that will run after any value is returned by the join point
  • After Throwing advice: If any exception thrown by the join point, this advice will be executed
  • Around advice: The method that will run before and after method invocation. It can also, choose to proceed to joint point call or to return its own value or throw an exception without proceeding to actual method call.

If you are using Springboot, the required libraries will be present in it, by default.

We can control the type of proxy by using the proxyTargetClass attribute. By default, it will be false.
