Implement Redis Cache in Springboot application


Cache is a temporary storage space to store data that your application needs frequently. This is preferred when your application expects low latency, but it has some bottleneck when querying original storage (like a database, API exposed by other service, etc).  

There are different types of caching available. Let’s discuss Redis cache techniques to integrate with a Springboot application.

To start with, you will need Redis installed on your machine. To install, follow the instructions from redis Official Documentation. 

For testing purposes, I will use H2 database as the main storage. So, we will need the following dependencies in pom.xml file:

To enable caching, annotate the Configuration class with @EnableCaching  

Include bean of RedisCacheManager if you need to customize cache configurations:

  • TTL (time to live): To set default time entries exist in cache. Once the time expires, the entry will be removed based on the eviction policy set in redis.conf 
  • Disable caching null values: Use disableCachingNullValues() 
  • Serialization/deserialization technique: Use serializeValuesWith() GenericJackson2JsonRedisSerializer is a common choice for JSON serialization.
  • To allow specific configurations for individual caches: Use withCacheConfiguration() that override the global settings. In the configuration below, Redis will have a TTL of 10 minutes. But we are setting 5 minutes for student cache.
package com.example.redis_cache_overview.config;

import org.springframework.cache.annotation.EnableCaching;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.data.redis.cache.RedisCacheConfiguration;
import org.springframework.data.redis.cache.RedisCacheManager;
import org.springframework.data.redis.connection.RedisConnectionFactory;
import org.springframework.data.redis.serializer.GenericJackson2JsonRedisSerializer;
import org.springframework.data.redis.serializer.RedisSerializationContext;

import java.time.Duration;

@Configuration
@EnableCaching
public class RedisConfig {

@Bean
public RedisCacheManager cacheManager(RedisConnectionFactory connectionFactory) {
    RedisCacheConfiguration defaultCacheConfig = RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofMinutes(10)); // Set a default TTL for caches
            

    return RedisCacheManager.builder(connectionFactory)
            .cacheDefaults(defaultCacheConfig)
            .withCacheConfiguration("StudentCache", defaultCacheConfig.entryTtl(Duration.ofMinutes(5))) // Cache-specific TTL
            .build();
}
}

Let’s create the service class where cache operations happen:

package com.example.redis_cache_overview.service;

import com.example.redis_cache_overview.dao.StudentRepository;
import com.example.redis_cache_overview.model.Student;
import io.lettuce.core.support.caching.RedisCache;
import org.aspectj.weaver.tools.cache.CacheStatistics;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.cache.CacheManager;
import org.springframework.cache.annotation.CacheEvict;
import org.springframework.cache.annotation.CachePut;
import org.springframework.cache.annotation.Cacheable;
import org.springframework.stereotype.Service;
import java.util.List;
import java.util.Optional;

@Service
public class RedisService {

    @Autowired
    StudentRepository studentRepository;

    public List<Student> getStudentDetail() {
        System.out.println("getting all student details");
        return studentRepository.findAll();
    }

    @Cacheable(value = "student", key = "#id")
    public Student getStudentDetailById(int id) {
        System.out.println("getting student by Id from DB:"+id);
        Optional<Student> student = studentRepository.findById(id);

        return student.orElse(null);
    }

    @CachePut(value = "student", key = "#student.id")
    public Student saveStudent(Student student) {
        System.out.println("Saving student detail to DB and pushing to cache "+student);
        return studentRepository.save(student);
    }

    @CacheEvict(value = "student", key = "#id")
    public void deleteStudentDetailById(int id) {
        System.out.println("deleting student by Id from DB and cache:"+ id);
        studentRepository.deleteById(id);
    }
}

In the above class, we are using below annotations on methods:

@Cacheable: It will check the cache content. For the required key, if a cache entry is available, it will return data without executing the method. If not, it will execute the method and make an entry in the cache for future use and return the data.

@CacheaPut: It will execute the method and update the cache with the latest details. This ensures the cache has up-to-date data.

@CacheEvict: It will execute the method and remove unwanted entries from the cache. 

Along with other properties, to log your application interaction with H2 database and cache, include the following properties in application.properties

logging.level.org.springframework.cache=TRACE
spring.jpa.show-sql=true

Let’s make some entries in the H2 database and cache during application startup:

package com.example.redis_cache_overview;

import com.example.redis_cache_overview.model.Student;
import com.example.redis_cache_overview.service.RedisService;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.ConfigurableApplicationContext;
import java.time.LocalDate;
import java.time.Month;
import java.util.List;

@SpringBootApplication
public class RedisCacheOverviewApplication {

 public static void main(String[] args) {
  ConfigurableApplicationContext run = SpringApplication.run(RedisCacheOverviewApplication.class, args);
  RedisService redisService = run.getBean(RedisService.class);

  Student joey = new Student(1,"Joey", LocalDate.of(1994, Month.SEPTEMBER, 21), LocalDate.of(2022, Month.AUGUST, 01));
  Student monica = new Student(2,"Monica", LocalDate.of(1996, Month.SEPTEMBER, 21), LocalDate.of(2024, Month.AUGUST, 01));
  redisService.saveStudent(joey);
  redisService.saveStudent(monica);
 }

}
