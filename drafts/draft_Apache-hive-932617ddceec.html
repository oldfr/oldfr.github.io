Apache hive 


Apache Hive is a data warehouse system that is distributed and fault-tolerant. This allows the analysis of huge amounts of data using SQL queries.

Hive Metastore(HMS) is a critical component that has a central repository of metadata used to analyze to make data driven decisions.

Hive is built on top of Apache Hadoop and supports storage on S3, ADLS, GS etc., though HDFS.


Features:

1. Hive-Server 2 (HS2)


  • The service that enables clients to execute queries against Hive. 
  • This supports multi-client concurrency and authentication. 
  • It is designed to provide better support for open API clients like JDBC and ODBC.

2. Hive Metastore Server (HMS)


  • This is a central repository of metadata for Hive tables and partitions in a relational database
  • It provides clients (including Hive, Impala and Spark) access to this information using the metastore service API

5. Hive Iceberg


Hive provides out of the box support for Apache Iceberg Tables, a cloud-native, high-performance open table format, via Hive StorageHandler.

7. Hive LLAP


interactive and subsecond SQL through Low Latency Analytical Processing (LLAP), introduced in Hive 2.0 that makes Hive faster by using persistent query infrastructure and optimized data caching

8. Query planner and Cost based Optimizer


Hive uses Apache Calcite’s cost based query optimizer (CBO) and query execution framework to optimize SQL queries.


9. Hive Replication


Hive supports bootstrap and incremental replication for backup and recovery.
