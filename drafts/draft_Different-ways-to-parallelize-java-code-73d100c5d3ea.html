Different ways to parallelize java code 


In today’s world parallelism is the common thing. We could complete the tasks in minimal time running them in parallel threads.

In java, there are many ways to do that

  1. ParallelStreams — After Java 8 version, we can use parallel streams to perform any tasks. Although it does not guarantee the parallel execution, we can use it whenever strict parallelism is not required
  2. ExecutorService- After Java 5, ExecutorService is available. It allow you to create threads based on requirement. For more details you can refer my article on executorService
  3. CompletableFuture — API allows to easily chain more calls with thenApply(), thenCompose() etc. It is thus more flexible than the simple Future returned by ExecutorService.submit(). If you didn’t specify an executor, it will use the common ForkJoinPool. You would have to pass an executor as second argument of supplyAsync() if you don't want the default executor.
