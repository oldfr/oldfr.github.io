Enable authentication and configure ActiveMQ in Java application

Hey There, as you might know, Apache ActiveMQ is the most popular open-source, multi-protocol, Java-based message broker. But, by default it does not have strict authentication to access from any application. It works without a username and password. You could configure it if required. Let's look at how we can do it.

Today, I will show you how we can enable authentication in ActiveMQ and connect to it from our Java application. You can download the ActiveMQ if not already present from the official documentation.

  • To enable authentication in public ActiveMQ

To enable authentication, you can use Simple Authentication Plugin in your activemq.xml file located in conf folder. Add below inside <broker> tag with required username and password.

activemq.xml:

<plugins>
   <simpleAuthenticationPlugin anonymousAccessAllowed="false">
     <users>
         <authenticationUser username="system" password="system" groups="users,admins"/>
         <authenticationUser username="test01" password="test01" groups="users,admins"/>
     </users>
   </simpleAuthenticationPlugin>
</plugins>

We can use any of these credentials to connect to ActiveMQ from our application. Any wrong credentials won’t work.

Set anonymousAccessAllowed = true, if you want to allow anonymous user access. Get more information on this here.

To update default credentials:

Among the above credentials, the one to be used by default, need to be provided in credentials.properties file as below:

activemq.username=test01
activemq.password=test01

To update credentials for web console access:

By default, web console access is using credentials admin/admin. To update it, add credentials in jetty-realm.properties file as below:

admin: system, admin
user: user, user

It is in the format:- username: password [,rolename...] with optional rolenames.

Whenever you access http://localhost:8161 , you need to give either username = admin and password= system (or ) username = user and password = user.
