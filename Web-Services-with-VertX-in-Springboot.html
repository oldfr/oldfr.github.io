Web Services with VertX in Springboot

Vert. x is a toolkit used for building reactive applications on the JVM using an asynchronous and non-blocking execution model. Vertx uses an event bus to communicate with different parts of the application passing the required information.

Today, we will see how to use Vertx for RESTful web services. Alternatively, many kinds of implementations are present for APIs, which include Spring REST, SOAP, and gRPC.

To use Vertx for web services, we need the below dependency in pom.xml file:

  <dependency>
   <groupId>io.vertx</groupId>
   <artifactId>vertx-web</artifactId>
   <version>4.1.0</version>
  </dependency>

In Vertx, we have verticles - the piece of code executed by Vert.x engine. To create any verticle, we need to implement the interface io.vertx.core.Verticle or io.vertx.core.AbstractVerticle which implements it.

If the verticle needs to perform a simple synchronous start-up task, then override start() method of the above class.

We can create a simple verticle as below:

import io.vertx.core.AbstractVerticle;

public class MainVerticle extends AbstractVerticle {
  public void start() {
    vertx.createHttpServer().requestHandler(req -> {
      req.response()
        .putHeader("content-type", "text/plain")
        .end("Hello from Vert.x!");
    }).listen(8080);
  }
}

In the above code, we are listening to API requests at the 8080 port. For all requests, we will get “Hello from Vert.x!” as a response.

Once we create a verticle, we need to deploy it in the application start-up to be able to invoke it. We can do that as below:

package com.example.vertxoverview.config;

import com.example.vertxoverview.verticle.HelloVerticle;
import com.example.vertxoverview.verticle.MainVerticle;
import io.opentracing.Tracer;
import io.vertx.core.Vertx;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import javax.annotation.PostConstruct;

@Configuration
public class MainConfig {

    @Autowired
    MainVerticle mainVerticle;

    @PostConstruct
    public void deployVerticle() {
        Vertx vertx = Vertx.vertx();
        vertx.deployVerticle(mainVerticle);
    }
}

Obviously, we don’t want to return the same response for all kinds of URIs. To customize this behavior and map specific endpoints for required tasks, use Router as a request handler.

The Router helps to create a number of endpoints with the required HTTP method, path, path parameters, body (in case of POST endpoint), and a handler for each of them.

package com.example.vertxoverview.verticle;

import io.vertx.core.AbstractVerticle;
import io.vertx.core.Promise;
import io.vertx.core.http.HttpServerOptions;
import io.vertx.ext.web.Router;
import io.vertx.ext.web.RoutingContext;
import org.springframework.stereotype.Component;

import java.io.FileNotFoundException;

@Component
public class MainVerticle extends AbstractVerticle {

    @Override
    public void start(Promise<Void> startPromise) throws Exception {

        vertx.createHttpServer(new HttpServerOptions())
                .requestHandler(getRouter())
                .listen(8080,asyncResult -> {
                    if (asyncResult.succeeded()) {
                        System.out.println("SUCCESS");
                        startPromise.complete();
                    }
                    else {
                        System.out.println("FAILED");
                        startPromise.fail(asyncResult.cause());
                    }
                });

    }

    private Router getRouter() throws FileNotFoundException {
        Router router = Router.router(vertx);

//        GET request
            router.get("/testing").handler(handler -> handler.response().end("Hello "));

//        GET request with path parameters
            router.get("/greet/:name").handler(handler -> {
                String name = handler.pathParam("name");
                handler.response().end(String.format("Hello %s ", name));
            });

//        POST request
            router.post("/test").handler(context ->
            {
                context.request().bodyHandler(System.out::println);
                context.response().end("done");
            });

        return router;
    }

}

If we want to reroute any API request to another endpoint we can do that using reroute() function in RoutingContext as below:

private Router getRouter() {
Router router = Router.router(vertx);

// re-route "/america" to "/india"
router.get("/america").handler(a -> {
            a.reroute("/india");
        });

router.get("/india").handler( i-> i.response().end("I love India"));
return router;
}

Events in Vertx:

Consider we are calling an API and if it is successful, we want to do some other tasks (e.g.:- place a message in Message Queue that the request has been completed). In that situation, we can use the event bus provided by Vertx engine.

For that, let's include another verticle that does the task of post API success tasks as below and deploy it (refer MainConfig at the top):

package com.example.vertxoverview.verticle;

import io.vertx.core.AbstractVerticle;
import org.springframework.stereotype.Component;

@Component
public class MQHandleVerticle extends AbstractVerticle {

    @Override
    public void start() throws Exception {
        vertx.eventBus().consumer("mqHandlerOnAPICompletion",msg -> {
            System.out.println("placing message in MQ");
            // place a success message in MQ with required details
           msg.reply("Success");
        });
    }
}

In the above code, Vertx creates a consumer with the name mqHandlerOnAPICompletion and a handler function. So, whenever a request for mqHandlerOnAPICompletion event raised in the application, it calls the handler function and replies to the caller with an appropriate message.

Below is an example code in route handler:

    private Router getRouter(){
        Router router = Router.router(vertx);

        // call API and place message in MQ after completion
        router.post("/student").handler(this::placeMessageInMq);
        return router;
    }

    private void createStudentRecAndplaceMessageInMq(RoutingContext ctx) {
//        perform API call task...
        vertx.eventBus().request("mqHandlerOnAPICompletion","", reply -> ctx.request().response().end((String) "Created student record. Placing message in MQ:"+reply.result().body()));
    }
