Pipeline Design pattern

Pipeline design pattern allows sequential data processing at different stages allowing modularity and reusability of code.

Consider we have N number of functions to be called one after the other and execution of one function depends on another function’s output. To simplify this implementation, we can go with the pipeline design pattern that generalizes each function into abstraction, with generic Input and Output types.

Benefits:

  • Decoupling: all stages will be loosely coupled making it easier to maintain
  • Reusability: A stage could be part of different pipelines
  • Extensibility: Adding a new stage would not need any major changes to the existing ones.

In this article, let’s implement a very simple pipeline where the input will be a string containing special characters, underscores, and numbers. We will create 3 methods that filter out underscores, special characters, and numbers in order. In the end, we will get a string with only characters. Although this scenario has much simpler approaches, we keep this example to make it easier to understand.

Create an interface with a method taking generic input and output:

package com.example.pipeline_design_pattern_overview.pipeline;

public interface BaseFunction<I,O> {

    public String getFunctionName(); // To get name of function implementing it
    public O apply(I input); 
}

Each stage will implement this interface and provide an implementation for apply() and getFunctionName() methods. Here, we have 3 classes implementing it:


public class RemoveSpecialChars implements BaseFunction<String, Object> {

    // implement apply() that takes String as input and outputs String
}

public class RemoveUnderScore implements BaseFunction<String, Object> {

    // implement apply() that takes String as input and outputs String
}


public class RemoveNumbers implements BaseFunction<String, Object> {

    // implement apply() that takes String as input and outputs char[] 
}

Create a component for pipeline creation, that includes a method taking BaseFunction implementation and stores in a list.

package com.example.pipeline_design_pattern_overview.model;

import com.example.pipeline_design_pattern_overview.pipeline.BaseFunction;
import org.springframework.stereotype.Component;

import java.util.ArrayList;
import java.util.List;

@Component
public class PipelineCreator {

    private static List<BaseFunction<String, Object>> handlers = new ArrayList<>();

    public PipelineCreator addHandler(BaseFunction<String, Object> handler) {
        handlers.add(handler);
        return this;
    }

    public List<BaseFunction<String, Object>> getHandlers() {
        return handlers;
    }
}

Create a bean to form a pipeline with all steps one by one. Here, the order is underscore, special characters, and the numbers:

package com.example.pipeline_design_pattern_overview.config;

import com.example.pipeline_design_pattern_overview.model.PipelineCreator;
import com.example.pipeline_design_pattern_overview.pipeline.impl.RemoveNumbers;
import com.example.pipeline_design_pattern_overview.pipeline.impl.RemoveSpecialChars;
import com.example.pipeline_design_pattern_overview.pipeline.impl.RemoveUnderScore;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class PipelineConfig {

    @Bean("PipelineStages")
    public PipelineCreator getPipeline() {
        return new PipelineCreator()
                .addHandler(new RemoveUnderScore())
                .addHandler(new RemoveSpecialChars())
                .addHandler(new RemoveNumbers());
    }
}

Service layer to invoke the pipeline and respond with the result:

package com.example.pipeline_design_pattern_overview.service;

import com.example.pipeline_design_pattern_overview.model.PipelineCreator;
import com.example.pipeline_design_pattern_overview.pipeline.BaseFunction;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.ArrayList;
import java.util.List;

@Service
public class PipelineService {

    @Autowired
    PipelineCreator pipelineCreator;

    public char[] execute(String input) {
        List<BaseFunction<String, Object>> handlers = pipelineCreator.getHandlers();

        var interResult = (Object)input;
        for(int handler = 0; handler < handlers.size(); handler++) {
            interResult = handlers.get(handler).apply((String) interResult);
        }
        return (char[]) interResult;
    }

}

Execution:

Let’s try with input → ab-(8)$9_UP+_F

Input is:ab-(8)$9_UP+_F
removing underscores from :ab-(8)$9_UP+_F
after removing underscores:ab-(8)$9UP+F
removing special chars from :ab-(8)$9UP+F
after removing special chars:ab89UPF
removing numbers from :ab89UPF
after removing numbers:abUPF

The final output will be → abUPF
